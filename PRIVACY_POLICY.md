# Privacy Policy

**Last Updated:** April 20, 2026

## Information We Collect

FlitPDF is designed to keep document processing local whenever possible.

The app may store limited local data such as:

- Recent file names and paths
- App preferences and lightweight usage state
- Optional account information if you choose to use Google Sign-In

We do not upload or store your document contents on our servers.

## How Information Is Used

Local data is used to:

- Provide core app functionality
- Remember recent activity and preferences
- Improve usability and stability

## Third-Party Services

Some optional features may rely on third-party services, including:

- Google Sign-In
- Firebase services used by authentication-related flows
- OS-level sharing integrations you choose to use

If you enable or use those services, their own terms and privacy policies also apply.

## Data Storage

- User documents remain on your device unless you explicitly share them
- Recent-file metadata and preferences can be cleared from within the app
- Private development credentials are not intended to be committed to the repository

## Permissions

FlitPDF may request permissions such as:

- Camera access for document scanning
- Storage or media access for reading and saving files
- Network access for optional online integrations

You can deny optional permissions, though some features may no longer work as expected.

## Open Source Repository

This repository is public for collaboration, but contributors should never commit personal data, private keys, keystores, or service credentials.
